# 女優ページ AV博士コメント 作成スキル

女優ページに表示する「AV博士のコメント」を、リサーチに基づいて生成し、DBの `actresses.comment` カラムに保存する。

---

## av博士キャラクター定義

write-say-commentsスキルと同一のキャラクター。以下を厳守する。

**話し方の特徴**
- 常体（だ・である）だが、硬すぎない。口語的なリズム感がある
- 自分の個人的な見解・反応を率直に語る（「個人的にドンピシャ」「正直やばい」「これは買いだ」）
- 比較・文脈で語る（「〇〇専属にしては珍しく」「このシリーズの中では」「このレーベルらしい」）
- 感情を抑えず、刺さったものは刺さったと言う
- 具体的な事実・エピソードを必ず1つ以上含める

**避けるべきパターン（AI文体）**
- 「〜は唯一無二の存在だ」
- 「〜は今後の活躍から目が離せない」
- 「〜にふさわしい逸材だ」
- 「〜と期待している」（抽象的な期待表明）
- 「〜は〇〇だ。今後の活躍に注目したい。」（2文で定型的に締める）
- 過度な体言止めの連続
- 同じ構文の繰り返し
- 意味が曖昧な表現（「数字がそのまま武器」のように、何がどう武器なのか不明確な文）

## 「エロさ」の判断基準

コメントで「エロい」を語るとき、以下のフレームワークを使うこと。的外れなことを書かないための指針。

### 興奮の本質は「体験の疑似再現」
読者は「この女優の作品を見たらどう感じるか」を知りたい。スペックや業績ではなく、**視聴体験として何が起きるか**を書く。

### 「ギャップ」の正しい使い方
- **弱いギャップ（避ける）**：「綺麗な顔で痴女をやる」「清楚な見た目なのにエロい」→ AV女優は基本綺麗。これはギャップではなく前提条件
- **強いギャップ（使える）**：
  - 素のインタビューと本番中の変容（普段ポンコツなのにカメラの前で豹変）
  - 見た目から想像できない体の反応（おとなしそうなのに感度が異常に高い、声の乱れ方が予想外）
  - 体型から予測できないプレイの質（小柄なのに騎乗位の腰使いがえぐい、華奢なのにパワフル）
- **ギャップの条件**：「ベースの印象」が確立されていて、かつ「反転が予想外」であること。視聴者がその女優の普段を知らない場合、同一シーン内の視覚的矛盾（例：童顔×激しいプレイ）として機能するかどうかで判断する

### ファンが実際に「エロい」と感じるポイント
1. **感度・リアクションのリアリティ** — 「本当に感じてるように見える」かどうか。作った喘ぎ声より、思わず漏れた声
2. **表情の変容** — 快感で理性が飛ぶ瞬間の表情。特に「目が落ちる」瞬間
3. **体の動き・腰使い** — 主体的に動く、本能的に反応している印象
4. **声・喘ぎ方の質感** — 声のトーン、乱れ方、息づかい
5. **変容のプロセス** — 最初は抑えていたのに徐々に崩れていく流れ（静→動）
6. **没入感（彼女感・疑似体験）** — POVやVRで「自分が体験している」感覚

### 的外れな表現（絶対に避ける）
- 「美人だからエロい」→ 美貌はエントリー条件。エロさの評価軸ではない
- 「綺麗な体」→ ビジュアル称賛であってエロさの評価ではない
- 「演技が上手い」→ 一般的な演技力ではなく「演技に見えないリアリティ」を語るべき
- 「〇〇賞を受賞」→ 業界賞は視聴体験の興奮とは無関係
- スペック羅列（B88/W59/H85）→ 数字だけでは感覚が伝わらない。「B88の柔らかい胸がパイズリで映える」のように体験に繋げる

## コメントのルール

### 長さ
3〜5文。記事の :::say ブロック（2〜4文）よりやや長めでOK。

### スマホ表示の考慮
コメントはスマホ表示で **4行まで** 表示され、残りは「続きを読む」で折りたたまれる。そのため **冒頭1〜2文で読者を惹きつける** 内容にすること。身体的スペックの羅列や前提説明から始めない。経歴のハイライト、意外なエピソード、端的な評価など「続きを読みたくなる」導入にする。

### コメントの目的
読者は「この女優のどこがエロいのか？何を見ればいいのか？」を知りたい。業界評論ではなく **おかずを選ぶときに役立つ情報** を提供する。

### 必須要素
コメントには以下のうち **2つ以上** を含めること：

1. **プレイスタイル・得意技**（「騎乗位の腰使いが別格」「淫語のアドリブ力がすごい」「感度が高くてリアクションがリアル」「潮吹きの量がえぐい」など）
2. **身体的魅力とそれが活きるジャンル**（「巨乳×小柄だからパイズリが映える」「引き締まった腹筋が体位モノで最高」「デカ尻が騎乗位で迫力満点」など、体型→どのジャンルで見るべきかを繋げる）
3. **バックグラウンド→エロさの理由**（「元ダンサーだから腰の動きが段違い」「元陸上部で鍛えた体幹がSEXに出てる」「元バレリーナだから柔軟性が異常」など、経歴がプレイにどう活きてるか）
4. **おすすめの見方・ジャンル**（「この人は人妻モノが一番ハマる」「VRで見ると没入感が段違い」「痴女モノでの本気度が違う」）
5. **性癖・本人の嗜好**（「本人がドM」「素でレズ好き」「本人がオタクだからコスプレの再現度が高い」など、本人の性癖や嗜好が作品に反映されてるケース）

### 禁止事項
- 品番（CID）をそのまま書かない
- プロフィール情報の単なる羅列（「B84・W58で身長159cm」だけで終わらない）
- リサーチで裏が取れない情報は書かない
- **業界評論家っぽい情報を主軸にしない**（受賞歴、ランキング順位、出演本数、メーカー遍歴は補足程度に留める。それ自体が興奮する情報ではない）
- 「〇〇賞を受賞」「年間ランキング〇位」で文を始めない

## 実行フロー

### 1. 対象の特定

ユーザーから以下のいずれかで対象を受け取る：
- 女優名（例：「三上悠亜のコメントを作って」）
- slug（例：「mikami-yua」）
- 条件指定（例：「コメントがない女優全員」「最近追加した女優」）

### 2. DBからプロフィール・作品データを取得

```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
SELECT a.id, a.name, a.slug, a.bust, a.waist, a.hip, a.height, a.birthday,
       a.blood_type, a.hobby, a.prefectures, a.debut_date, a.comment,
       COUNT(aw.work_id) as work_count
FROM actresses a
LEFT JOIN actress_work aw ON a.id = aw.actress_id
WHERE a.slug = '{slug}'
GROUP BY a.id
"
```

ジャンル情報も取得：
```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
SELECT g.name, COUNT(DISTINCT w.id) as cnt
FROM genres g
INNER JOIN work_genre wg ON g.id = wg.genre_id
INNER JOIN works w ON wg.work_id = w.id
INNER JOIN actress_work aw ON w.id = aw.work_id
INNER JOIN actresses a ON aw.actress_id = a.id
WHERE a.slug = '{slug}'
GROUP BY g.id ORDER BY cnt DESC LIMIT 10
"
```

### 3. WebSearchでリサーチ

以下の観点で検索する（並列Agent推奨、model: "sonnet"）：

- **プレイスタイル・得意ジャンル**（何が上手いか、どんなプレイで評価されてるか）
- **身体的特徴のエロさ**（ファンが語る体の魅力、作品で活きてるポイント）
- **バックグラウンド**（前職・部活・趣味で、体や演技に影響してるもの）
- **本人の性癖・嗜好**（インタビューで語った性的嗜好、素の性格が作品に出てるケース）
- **ファンのおすすめジャンル・シリーズ**（「この人は〇〇で見るべき」的な評価）
- **代表作・人気シリーズ**（何が売れたか、なぜ売れたか）
- **引退・復帰情報**（現役かどうかの確認用）

### 4. コメント生成

リサーチ結果 + DBのプロフィール・ジャンルデータを統合し、av博士キャラクターでコメントを書く。

**生成前チェックリスト**：
- [ ] 「この女優のどこがエロいか」が伝わるか
- [ ] プレイスタイル・身体的魅力・おすすめジャンルのうち2つ以上あるか
- [ ] 受賞歴やランキングが主役になっていないか（補足程度ならOK）
- [ ] AI文体の定型句が混ざっていないか
- [ ] プロフィールの羅列になっていないか
- [ ] 品番が含まれていないか

### 5. DBに保存

```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
UPDATE actresses SET comment = '{コメント}' WHERE id = {id};
"
```

保存後、キャッシュをクリア：
```bash
php /Users/kaikyotaro/repository/video-meikan/meikan/batch/clear_cache.php
```

### 6. 確認

保存後、ユーザーにコメント内容を提示して確認を求める。修正があれば反映する。

## 一括実行モード

「コメントがない女優全員」のように複数対象の場合：

1. 対象リストを取得：
```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
SELECT id, name, slug FROM actresses WHERE comment IS NULL ORDER BY id
"
```

2. 5人ずつ並列Agentでリサーチ（model: "sonnet"でコスト効率化）
3. コメントを一括生成・保存
4. 全件完了後にキャッシュクリア

## 良いコメント例

```
騎乗位の腰使いが業界でも別格で、「スパイダー騎乗位を体現化・定着させた」とAV監督に名指しされてる。趣味のベリーダンスで鍛えた体幹がそのまま腰の動きに出てて、特に騎乗位モノを見ると納得する。セリフを覚えるのが苦手だったらしく、そこからアドリブを磨いた結果、淫語の即興力が異常に高い。痴女モノで見るのが一番この人の良さが出る。
```
→ 得意プレイ（騎乗位）＋バックグラウンド（ベリーダンス→体幹）＋おすすめジャンル（痴女モノ）が繋がってる

```
169cmでバレー強豪校出身、引き締まった体がそのまま正常位や立ちバックで映える。感度が高くてリアクションが大げさじゃないのがリアルで、特にS1時代の作品は「何を撮っても成立する」安定感があった。本人が「普通の女の子のときの私はポンコツ」と泣きながら言ったのが妙に記憶に残ってるが、カメラの前では別人になるタイプ。
```
→ 身体的魅力（長身＋バレー体型→体位モノ）＋プレイスタイル（リアルなリアクション）＋おすすめ時期（S1時代）

```
155cmの小柄な体がVRだと没入感を増幅させてて、VR部門で年間1位を2回取ってるのはこの人だけ。「雪山遭難VR」「だいしゅきホールドVR」みたいにシチュエーション特化の作品が多くて、小さい体で抱きついてくる感じがたまらない。清楚系の顔とのギャップもあって、「ボクの彼女」感が欲しいならVRから入るのがおすすめ。
```
→ 身体的魅力（小柄→VRの没入感）＋おすすめの見方（VRから入れ）＋具体的な作品の魅力が伝わる
