# 女優ページ AV博士コメント 作成スキル

女優ページに表示する「AV博士のコメント」を、リサーチに基づいて生成し、DBの `actresses.comment` カラムに保存する。

---

## av博士キャラクター定義

write-say-commentsスキルと同一のキャラクター。以下を厳守する。

**話し方の特徴**
- 常体（だ・である）だが、硬すぎない。口語的なリズム感がある
- 自分の個人的な見解・反応を率直に語る（「個人的にドンピシャ」「正直やばい」「これは買いだ」）
- 比較・文脈で語る（「〇〇専属にしては珍しく」「このシリーズの中では」「このレーベルらしい」）
- 感情を抑えず、刺さったものは刺さったと言う
- 具体的な事実・エピソードを必ず1つ以上含める

**避けるべきパターン（AI文体）**
- 「〜は唯一無二の存在だ」
- 「〜は今後の活躍から目が離せない」
- 「〜にふさわしい逸材だ」
- 「〜と期待している」（抽象的な期待表明）
- 「〜は〇〇だ。今後の活躍に注目したい。」（2文で定型的に締める）
- 過度な体言止めの連続
- 同じ構文の繰り返し
- 意味が曖昧な表現（「数字がそのまま武器」のように、何がどう武器なのか不明確な文）

---

## コメントのルール

### 長さ
3〜5文。記事の :::say ブロック（2〜4文）よりやや長めでOK。

### 必須要素
コメントには以下のうち **2つ以上** を含めること：

1. **具体的な事実・数字**（受賞歴、ランキング順位、作品数、売上実績）
2. **経歴のストーリー**（デビュー経緯、メーカー遍歴、引退・復帰、前職）
3. **身体的特徴と作品での活かされ方**（スタイルがどのジャンル・シリーズに繋がっているか）
4. **業界内での位置づけ**（他の女優との比較、メーカーでの立ち位置）
5. **個人的エピソード**（インタビュー発言、趣味と作品の関連、意外な経歴）

### 禁止事項
- 品番（CID）をそのまま書かない
- プロフィール情報の単なる羅列（「B84・W58で身長159cm」だけで終わらない）
- リサーチで裏が取れない情報は書かない

## 実行フロー

### 1. 対象の特定

ユーザーから以下のいずれかで対象を受け取る：
- 女優名（例：「三上悠亜のコメントを作って」）
- slug（例：「mikami-yua」）
- 条件指定（例：「コメントがない女優全員」「最近追加した女優」）

### 2. DBからプロフィール・作品データを取得

```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
SELECT a.id, a.name, a.slug, a.bust, a.waist, a.hip, a.height, a.birthday,
       a.blood_type, a.hobby, a.prefectures, a.debut_date, a.comment,
       COUNT(aw.work_id) as work_count
FROM actresses a
LEFT JOIN actress_work aw ON a.id = aw.actress_id
WHERE a.slug = '{slug}'
GROUP BY a.id
"
```

ジャンル情報も取得：
```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
SELECT g.name, COUNT(DISTINCT w.id) as cnt
FROM genres g
INNER JOIN work_genre wg ON g.id = wg.genre_id
INNER JOIN works w ON wg.work_id = w.id
INNER JOIN actress_work aw ON w.id = aw.work_id
INNER JOIN actresses a ON aw.actress_id = a.id
WHERE a.slug = '{slug}'
GROUP BY g.id ORDER BY cnt DESC LIMIT 10
"
```

### 3. WebSearchでリサーチ

以下の観点で検索する（並列Agent推奨、model: "sonnet"）：

- デビュー作の詳細（メーカー、レーベル、反響）
- 代表作・人気シリーズ
- 所属メーカー・レーベルの遍歴
- 受賞歴・ランキング実績
- 業界での立ち位置・特徴的なエピソード
- ファンや業界からの評価
- 引退・復帰情報

### 4. コメント生成

リサーチ結果 + DBのプロフィール・ジャンルデータを統合し、av博士キャラクターでコメントを書く。

**生成前チェックリスト**：
- [ ] 具体的な事実・エピソードが2つ以上入っているか
- [ ] 曖昧な表現がないか（「〇〇がそのまま武器」等）
- [ ] AI文体の定型句が混ざっていないか
- [ ] プロフィールの羅列になっていないか
- [ ] 品番が含まれていないか

### 5. DBに保存

```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
UPDATE actresses SET comment = '{コメント}' WHERE id = {id};
"
```

保存後、キャッシュをクリア：
```bash
php /Users/kaikyotaro/repository/video-meikan/meikan/batch/clear_cache.php
```

### 6. 確認

保存後、ユーザーにコメント内容を提示して確認を求める。修正があれば反映する。

## 一括実行モード

「コメントがない女優全員」のように複数対象の場合：

1. 対象リストを取得：
```bash
mysql -u root --socket=/tmp/mysql_meikan.sock video_meikan -e "
SELECT id, name, slug FROM actresses WHERE comment IS NULL ORDER BY id
"
```

2. 5人ずつ並列Agentでリサーチ（model: "sonnet"でコスト効率化）
3. コメントを一括生成・保存
4. 全件完了後にキャッシュクリア

```

```
